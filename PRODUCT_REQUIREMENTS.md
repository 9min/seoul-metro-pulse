# Product Requirements Document (PRD): Seoul Metro Pulse

## 1. 프로덕트 개요 (Product Vision)
**"도시의 혈관을 흐르는 맥박을 시각화하다."**
일반적인 지도 API 위에 표시되는 뻔한 노선도를 완전히 탈피합니다. 서울 지하철 네트워크 전체를 거대한 '거미줄(신경망)'처럼 재구성하고, 현재 운행 중인 수백 대의 실제 열차들이 역과 역 사이를 빛의 입자(Pulse)처럼 끊김 없이 부드럽게 흘러가는 모습을 감상할 수 있는 **'초실감형 실시간 관제 대시보드'**입니다.

## 2. 상세 기능 명세 (Feature Breakdown)
* **라이브 펄스 네트워크 맵 (The Live Pulse Map):** 수백 개의 빛나는 입자(현재 운행 중인 열차들)가 선을 따라 살아 움직이듯 부드럽게 이동합니다. 시맨틱 줌(Semantic Zoom)을 적용하여, 줌아웃 시에는 노선의 흐름만 조망하고 줌인 시에는 역 이름과 열차의 상세 정보가 페이드인됩니다.
* **스테이션 인스펙터 (Station Inspector):** 역을 클릭하면 화면 전체가 어두워지며 해당 역과 연결된 상/하행 열차들만 하이라이트됩니다. 우측 패널에는 실제 전광판과 같은 실시간 도착 예정 타이머가 표시됩니다.
* **트레인 트래커 (Train Tracker):** 특정 열차(입자)를 클릭하면 카메라가 열차의 궤적을 자동으로 추적(Follow)하며, 다음 역까지 남은 도착 시간을 프로그레스 바 형태로 제공합니다.

---

## 3. 핵심 기술 및 아키텍처 전략 (Core Architecture)

사용자를 압도하는 이 시각적 경험을 구현하기 위해, 무거운 상용 라이브러리를 배제하고 브라우저 네이티브 기술과 공간 연산 알고리즘을 극한으로 활용합니다.

### 3.1. 데이터 소스: 완전한 무과금 공공 API 파이프라인 (Data Source)
대한민국은 공공 데이터가 세계 최고 수준으로 개방되어 있습니다. 특히 '서울/수도권 지하철' 데이터는 완벽하게 무료로 제공되므로 백엔드 서버 비용이 0원에 수렴합니다.

* **실시간 열차 위치 API:** **'서울열린데이터광장'**의 `서울시 지하철 실시간 열차 위치 정보 API`를 사용합니다. (하루 1,000건 무료 제공되며, 포트폴리오용으로 충분하고 필요시 제한을 늘릴 수 있습니다.)
* **지하철역 좌표 및 노선 데이터:** 공공데이터포털에서 제공하는 `전국 역사 좌표 데이터`를 CSV나 JSON 형태로 무료 다운로드하여 정적 에셋으로 활용합니다.

### 3.2. 렌더링 아키텍처: 지도 API의 종말과 직접 렌더링 (The Architecture)
주니어 개발자들은 보통 카카오맵 API를 띄우고 그 위에 마커를 올립니다. 하지만 14년 차 아키텍트인 구민님은 무거운 상용 지도를 버리고 **'데이터 시각화(Data Visualization) 엔진'**을 클라이언트 단에 직접 구축합니다. 

* **Node-Link 구조화:** 다운로드한 지하철역 데이터를 프론트엔드에서 즉시 연산할 수 있는 JSON 그래프 객체로 가공합니다.
  * **Nodes:** 역 데이터 (역 이름, 고유 ID, X/Y 좌표)
  * **Links:** 역과 역 사이의 연결선 (출발역 ID, 도착역 ID, 호선 색상)
* **Canvas / WebGL 렌더링:** 무거운 DOM 대신 `PixiJS`나 순수 `Canvas API`를 사용하여 화면에 수백 개의 역(점)과 노선(선)을 단숨에 그려냅니다. 무거운 맵 타일(Map Tile)을 불러오는 네트워크 비용이 0원이 되며, 화면을 줌/팬(Zoom/Pan) 할 때 60fps의 압도적인 퍼포먼스를 보장합니다.

### 3.3. 기술적 하이라이트: 공간 보간 알고리즘 (The "Kick")
서울시 API는 실시간이라고 하지만 사실 약 10초~15초 주기로 데이터가 업데이트 됩니다. 즉, 화면의 열차가 10초마다 뚝뚝 끊기며 순간이동을 한다는 치명적인 약점이 있습니다.

* **클라이언트 추측 항법 (Dead Reckoning & Interpolation):** 이 거친 데이터를 프론트엔드가 수학적으로 보정합니다. 이전 API 응답 위치와 현재 위치, 그리고 역 간의 거리를 계산하여 10초의 데이터 공백 기간 동안 열차의 움직임을 유추합니다. 이를 통해 서버 데이터의 한계를 넘어 60fps로 매끄럽게 선로를 미끄러지는 시네마틱한 모션을 창조해 냅니다.
