# PRD - Seoul Metro Pulse

## 1. 프로젝트 개요

**"도시의 혈관을 흐르는 맥박을 시각화하다."**

서울 지하철 네트워크 전체를 거대한 신경망처럼 재구성하고, 현재 운행 중인 수백 대의 실제 열차들이 역과 역 사이를 빛의 입자(Pulse)처럼 끊김 없이 부드럽게 흘러가는 모습을 감상할 수 있는 **초실감형 실시간 관제 대시보드**이다.

일반적인 지도 API 위에 표시되는 뻔한 노선도를 탈피하여, 무거운 상용 지도를 배제하고 PixiJS 기반의 데이터 시각화 엔진을 클라이언트에 직접 구축한다.

## 2. 목표 사용자

- **주 대상**: 채용 담당 CTO / 기술 리더
- **용도**: 개발자 포트폴리오 프로젝트
- **기대 효과**: 프론트엔드 아키텍처 설계 능력, 실시간 데이터 처리, Canvas/WebGL 렌더링 역량을 한눈에 증명

## 3. 핵심 기능

- [ ] **라이브 펄스 네트워크 맵 (The Live Pulse Map)**: 수백 개의 빛나는 입자(열차)가 노선을 따라 부드럽게 이동하는 실시간 시각화. 시맨틱 줌(Semantic Zoom) 적용 — 줌아웃 시 노선 흐름 조망, 줌인 시 역 이름과 열차 상세 정보 페이드인.
- [ ] **스테이션 인스펙터 (Station Inspector)**: 역 클릭 시 화면 전체가 어두워지며 해당 역 연결 열차만 하이라이트. 우측 패널에 실시간 도착 예정 타이머(전광판 스타일) 표시.
- [ ] **트레인 트래커 (Train Tracker)**: 열차(입자) 클릭 시 카메라가 열차 궤적을 자동 추적(Follow). 다음 역까지 남은 도착 시간을 프로그레스 바로 제공.

## 4. 사용자 스토리

- CTO로서 포트폴리오 사이트에서 이 대시보드를 열어 **프론트엔드 렌더링 성능과 아키텍처 역량을 즉시 확인**하고 싶다.
- 사용자로서 서울 지하철 전체 노선을 한눈에 조망하고, **특정 역이나 열차를 클릭하여 상세 정보**를 확인하고 싶다.
- 사용자로서 열차가 10초마다 뚝뚝 끊기지 않고 **60fps로 매끄럽게 이동하는 모션**을 감상하고 싶다.

## 5. 기술 스택

- **프론트엔드**: Vite + React + TypeScript
- **렌더링 엔진**: PixiJS (Canvas/WebGL)
- **상태 관리**: Zustand
- **CSS**: Tailwind CSS
- **린트/포매팅**: Biome
- **테스트**: Vitest
- **배포**: Vercel
- **데이터 소스**: 서울메트로 SMSS (1~8호선) + 서울열린데이터광장 공공 API (9호선)

## 6. 핵심 기술 아키텍처

### 6.1. 데이터 소스: 무과금 공공 API

- **1~8호선 실시간 열차 위치**: 서울메트로 SMSS (`smss.seoulmetro.co.kr`) HTML 스크래핑. Vercel Edge Function (`api/trains.ts`)이 HTML을 파싱하여 열차 위치 데이터로 변환한다. 폴링 주기 5초.
- **9호선 실시간 열차 위치**: 서울열린데이터광장 `서울시 지하철 실시간 열차 위치 정보 API`. Mixed Content/CORS 우회를 위해 Vercel Edge Function (`api/line9.ts`)을 서버사이드 프록시로 경유한다. 폴링 주기 30초.
- **지하철역 좌표/노선 데이터**: 공공데이터포털 `전국 역사 좌표 데이터`를 JSON 정적 에셋으로 활용

### 6.2. 렌더링: 지도 API 없는 직접 렌더링

- **Node-Link 그래프 구조**: 역(Nodes) + 연결선(Links) JSON 그래프
  - Nodes: 역 이름, 고유 ID, X/Y 좌표
  - Links: 출발역 ID, 도착역 ID, 호선 색상
- **PixiJS 렌더링**: DOM 대신 WebGL/Canvas로 수백 개의 역과 노선을 고성능 렌더링
- **줌/팬**: 60fps 보장

### 6.3. 클라이언트 추측 항법 (Dead Reckoning & Interpolation)

API의 업데이트 주기 한계(SMSS 5초, 9호선 30초)를 극복하기 위해, 이전/현재 API 응답 위치와 역 간 거리를 기반으로 열차 움직임을 수학적으로 보간하여 60fps 매끄러운 모션을 구현한다.

## 7. 비기능 요구사항

- **성능**: 60fps 렌더링, 수백 개 열차 동시 애니메이션
- **보안**: 공공 API 키 환경변수 관리, 사용자 입력 검증
- **접근성**: 키보드 탐색 지원, 색상 대비 기준 준수
- **반응형**: 데스크탑 우선 (포트폴리오 용도), 태블릿 대응 *(추론)*

## 8. 마일스톤

- **v0.1 (MVP)**: 정적 노선도 렌더링 + 줌/팬 + 역 데이터 표시
- **v0.2**: 실시간 열차 위치 API 연동 + 기본 입자 이동
- **v0.3**: 클라이언트 보간 알고리즘 + 60fps 매끄러운 모션
- **v0.4**: 스테이션 인스펙터 + 트레인 트래커
- **v1.0**: 시맨틱 줌 + 다크 테마 + 최종 폴리싱
